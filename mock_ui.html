Deadline Buddy — Wireframes

Low-fidelity static screens covering the main flow · Windows / macOS / mobile · v1.0 · 2026-07-06

Flow: Pool list → Create pool → Pool detail (paste Jira link to add ticket) → capacity-blocked state → overdue continuous red warning.

1. Pool list (home screen)

Sidebar and card grid both show live capacity ("used / capacity") and rollups of warning/overdue counts. Pools at or over capacity are flagged red so the user knows before opening them.

2. Create pool

Capacity and default warning lead time (≤120 min) are set at pool creation, per the requirement that lead time is user-configurable.

3. Pool detail — add ticket via Jira link

Adding a ticket only needs a pasted Jira URL — the backend resolves title, description, and deadline. Status badges reflect Normal / Warning / Overdue in place, no separate screen needed.

4. Capacity-blocked state

Server rejects the add with 409 PoolCapacityExceeded; the UI disables the add control and surfaces the reason instead of failing silently.

5. Overdue ticket — continuous red warning

This is the visual expression of the backend's repeating TicketOverdue Socket.IO event — the client keeps the red state rendered as long as it keeps receiving/holding it, and clears it only on TicketResolved.
